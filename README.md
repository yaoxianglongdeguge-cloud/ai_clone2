我打算把这个包做成一个可以批量复制产生agent的包

以下为文件架构

//本包

main.py  //接线员


--ai clone //agent的模板文件，含有agent需要的基本文件
      --clone类  //agent的模板，定义agent的类
      --des类    //agent工具的描述
      --tools类  //agent工具
        
—生产程序  //根据人设和功能创建agent实例，并且准备好基本文件

—记忆概括ai //提取agent的记忆进行概括。上一次概括后，原记忆会暂时储存，直到下一次概括。不过这里只实现类。

—入出des tools ai  //写入和删除tools的描述。这里只实现类




//生产的ai

--实例ai  //把程序从管理ai进入改成进入一个接线主程序，让所有agent平级，防止文件路径管理混乱
     --主程序  
     --des.txt  //工具描述，每次程序启动都会读取。
     --tools.py  //工具函数定义
     --memory.txt //记忆功能
     --memoryed.txt  //上一次记忆
     --role.txt  //人设

     --memory_ai //记忆概括ai在该文件夹的实例化
     --des_ai  //入出des ai实例化





ai有自己特殊文件，而自己的文件会在实例化的时候绑定，绑定的方法是，通过ai所在的文件夹名称，直接路径绑定
而主程序和ai运行程序分离，每次运行ai运行函数之后再实例化ai，而在主程序调用ai的运行函数时需要传入ai所在文件夹名称
这个名称必须和ai名称相同

ai用自己所在的文件夹名称锚定自己。函数工具用所在文件名称锚定自己。

打印回复。函数导入包(除了标准库)等最好都在函数内完成，方便模块化管理

sys路径加入只用在入口程序
